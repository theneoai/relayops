# 自动驾驶多传感器融合技术规范 v2.1

## 1. 概述

本文档定义了自动驾驶系统中多传感器融合的技术规范，包括LiDAR、摄像头、毫米波雷达和IMU/GNSS的数据融合策略。

## 2. 传感器配置

### 2.1 LiDAR（激光雷达）
- **型号**: 128线机械式LiDAR
- **探测距离**: 200m @ 10%反射率
- **角分辨率**: 0.1° (水平) × 0.2° (垂直)
- **点云密度**: ≥ 240万点/秒
- **帧率**: 10Hz / 20Hz 可选
- **精度**: ±3cm

### 2.2 摄像头
- **数量**: 8路（前视×2 + 侧视×4 + 后视×1 + 环视×1）
- **分辨率**: 1920×1200 @ 30fps
- **动态范围**: HDR 120dB
- **视场角**: 前视120° / 侧视80° / 后视100°
- **色彩空间**: RGB + IR

### 2.3 毫米波雷达
- **数量**: 5路（前向长距×1 + 角雷达×4）
- **前向雷达**: 探测距离300m，角度分辨率2°
- **角雷达**: 探测距离100m，角度分辨率4°
- **测速精度**: ±0.1km/h

### 2.4 IMU/GNSS
- **IMU采样率**: 200Hz
- **GNSS**: RTK-GPS + 北斗 + GLONASS
- **定位精度**: 厘米级（RTK模式下）
- **航向角精度**: 0.1°

## 3. 时间同步

所有传感器必须严格时间同步，要求：
- **同步精度**: < 1ms
- **同步方式**: PTP (Precision Time Protocol) + GPS授时
- **时间戳**: Unix时间戳，微秒精度
- **异常处理**: 超过5ms时间差的帧标记为"不同步"，丢弃或降级使用

## 4. 外参标定

### 4.1 标定矩阵
每个传感器相对于车辆坐标系（后轴中心，X向前，Y向左，Z向上）的变换矩阵：
- **旋转矩阵 R**: 3×3 正交矩阵
- **平移向量 t**: 3×1 (单位：米)
- **齐次变换矩阵 T**: 4×4

### 4.2 标定精度要求
- **LiDAR-Camera**: 重投影误差 < 3像素
- **LiDAR-Radar**: 距离对齐误差 < 0.5m
- **Camera-Camera**: 相邻摄像头重叠区域特征点匹配误差 < 2像素
- **IMU-LiDAR**: 时间对齐误差 < 5ms

### 4.3 在线标定监测
- 监测频率: 每100帧检查一次
- 监测指标: 静态场景下的融合一致性
- 异常阈值: 融合误差超过标定值的2倍触发重标定

## 5. 融合策略

### 5.1 数据级融合（早期融合）
- **适用场景**: 高精度3D目标检测
- **方法**: 将Camera的RGB特征投影到LiDAR点云BEV空间
- **优点**: 保留最多原始信息
- **缺点**: 计算量大，对传感器故障敏感

### 5.2 特征级融合（中期融合）
- **适用场景**: 实时目标检测与跟踪
- **方法**: 各传感器独立提取特征，在特征空间融合
- **融合权重**: 根据传感器置信度动态调整
  - 晴天白天: Camera权重 0.6, LiDAR权重 0.4
  - 雨天/夜间: Camera权重 0.3, LiDAR权重 0.6, Radar权重 0.1
  - 大雾: LiDAR权重 0.3, Radar权重 0.7

### 5.3 决策级融合（晚期融合）
- **适用场景**: 安全关键决策（AEB、避障）
- **方法**: 各传感器独立做决策，投票/加权融合
- **规则**: 任一传感器触发安全动作即执行（fail-safe）

## 6. 故障降级策略

| 故障传感器 | 降级模式 | 影响功能 | 限制速度 |
|-----------|---------|---------|---------|
| LiDAR | Camera + Radar融合 | 3D检测降级为2D+深度估计 | ≤ 60km/h |
| Camera | LiDAR + Radar融合 | 红绿灯/车道线识别降级 | ≤ 40km/h，仅高速 |
| Radar | LiDAR + Camera融合 | 速度测量精度下降 | 正常 |
| IMU/GNSS | 轮速计+LiDAR里程计 | 长距离定位漂移 | ≤ 80km/h |
| 双传感器故障 | 请求接管 | 自动驾驶不可用 | 立即停车 |

## 7. 性能指标

- **目标检测**: mAP ≥ 85%（3D），mAP ≥ 92%（BEV）
- **目标跟踪**: MOTA ≥ 85%
- **融合延迟**: ≤ 50ms（端到端）
- **CPU占用**: ≤ 30%（x86_64，8核）
- **内存占用**: ≤ 4GB

## 8. 安全要求

- 符合ISO 26262 ASIL-D等级
- 融合算法必须具备故障检测机制
- 传感器数据有效性检查（范围、连续性、一致性）
- 单一传感器故障不得导致危险输出
